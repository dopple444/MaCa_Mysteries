## Data Access Notes

- Public catalog/detail must expose only public-safe metadata and active product information.
- Host controls must remain spoiler-safe by default; admin pages are trusted spoiler-heavy views.
- Player `/play` must only expose content visible to the current guest, character assignment, active rounds, reveal state, and unlock state.
- Conditional cards/evidence/media must stay hidden until a matching party unlock event exists for the current guest/scope.
- Character tools may expose player-facing codes only to the assigned tool holder; code attempts should store hashes and return generic failure reasons.
- Host conditional activity should remain spoiler-safe by default and must never expose raw access codes.
- Checkout flow must remain provider-safe when credentials are missing and must use idempotent webhooks for fulfillment.
- Payment maintenance cancels stale unpaid orders as `CANCELLED` and reruns paid fulfillment idempotently.
- Outbound messages are queued locally; email delivery supports console dry-run and Resend provider adapters, while SMS delivery remains pending.
- Email verification and password reset use signed links and queued email delivery instead of long-lived token rows.
